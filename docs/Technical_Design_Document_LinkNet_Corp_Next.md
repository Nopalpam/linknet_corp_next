## 4. System Architecture

The current architecture is a multi-application workspace. The backend exposes REST APIs under `/api/v1` and health/root endpoints outside the prefix. The admin frontend is a protected CMS dashboard. The public web app renders CMS-driven pages, news, careers, events, dynamic forms, and market data. A separate filemanager service exposes an S3-backed API under `/api` with optional API key authentication.

| Layer | Actual implementation |
| --- | --- |
| Client/Admin | frontend/src/app Next.js app, AuthContext, service layer with credentials: include, Bearer token support, CSRF header support. |
| Client/Public | web/app Next.js app, next-intl middleware, cmsApi/formsApi/eventsApi, visitor tracker, local API routes for Yahoo Finance and form proxy. |
| API | backend/src/server.ts Express app with security, CORS, CSRF, rate limit, activity logging, route modules, global error handler. |
| Persistence | backend/prisma/schema.prisma models and migrations; Prisma client singleton under backend/src/config/database.ts. |
| Storage | Uploads served from UPLOAD_DIR for local; S3 and Azure service implementations found; file metadata persisted in File/Folder tables. |
| Async/Queue | Bull + Redis for activity/form dispatch when REDIS_ENABLED=true; fallback direct logging for activity logger. |

### 4.1 Frontend-Backend Communication

- Admin frontend uses `NEXT_PUBLIC_API_URL`, API prefix `/api/v1`, `credentials: include`, Authorization bearer token from localStorage/cookie, and `X-CSRF-Token` from cookie for unsafe methods.
- Public web uses `NEXT_PUBLIC_API_URL` or server-side `API_INTERNAL_URL` for CMS content, form modules, events, menu, settings, labels, and news.
- Next.js web API routes proxy Yahoo Finance data and enterprise form submission.

### 4.2 Authentication Flow

- Login receives email/password, applies business-hour middleware and login rate limiter, checks lockout/status/password expiry, verifies bcrypt hash, optionally requires MFA, then issues access and refresh JWTs.
- Refresh token is persisted as SHA-256 hash in `refresh_tokens` and rotated on refresh.
- Access token may be sent via Authorization header or `auth_token` HttpOnly cookie. Cookie-authenticated unsafe API methods require matching `X-CSRF-Token` and `csrf_token` cookie.
- RBAC middleware checks role/permission before protected CMS operations.

### 4.3 Request Flow

`Client request -> HTTPS redirect middleware -> Helmet/custom security headers -> request id -> HTTP logger -> CORS -> body parser/cookie parser -> CSRF middleware on /api -> compression -> health/static handling -> no-store + rate limiter + activity logger -> route middleware/controller -> Prisma/service/storage/external integration -> global error handler`.

### 4.4 Deployment Overview

Docker multi-stage builds are present for all four deployable components. Backend Kubernetes manifests include Deployment, Service, HPA, probes, securityContext, ConfigMap/Secret references, ingress TLS, and Azure Key Vault CSI examples. No complete docker-compose file was found in the repository root.

### 9.1 API Conventions

| Convention | Value |
| --- | --- |
| Backend base URL | API prefix defaults to /api/v1; root / and /health also exist. |

## 15. Technical Risks & Recommendations

| Severity | Finding | Recommendation |
| --- | --- | --- |
| High | CMS contact export route conflict | In backend/src/routes/cms/contactus.routes.ts, GET /export is declared after GET /:id, so /api/v1/cms/contactus/export may be captured as id. Move /export before /:id. |
| High | Frontend/backend settings endpoint mismatch | frontend settings service references /cms/settings/key/:key and /cms/settings/bulk-update, while backend defines /cms/settings/:key and /cms/settings/update-group. Align service or backend routes before SIT/UAT. |
| High | Frontend report grid-items endpoint mismatch | frontend report service references /cms/report-types/:id/grid-items and grid-items/update-order; backend report.routes.ts does not implement these endpoints. |
| Medium | JWT returned to client and localStorage usage | Backend sets HttpOnly cookies but also returns tokens in JSON, and admin frontend stores tokens in localStorage. This increases impact of XSS. Prefer cookie-only or memory-only access token strategy. |
| Medium | Storage abstraction inconsistency | S3 service is implemented, but storage/s3.storage.ts provider is documented as not yet implemented/throws. Complete provider or prevent STORAGE_DRIVER=s3 from selecting incomplete abstraction. |
| Medium | File malware scanning is opportunistic | If scanner/ClamAV is unavailable, code logs warning and allows file. For strict security review, enforce scanner availability in production. |
| Medium | Standalone filemanager API key optional | filemanager auth middleware disables protection if API_KEY is not configured. Production deployment must configure API_KEY or isolate the service. |
| Medium | Duplicate event admin aliases | Event routes expose both /cms/events and /admin/events aliases. Maintain only one canonical path or document both to avoid inconsistent frontend/API usage. |
| Medium | Development rate limit default | Global rate limit can be disabled outside production. Ensure RATE_LIMIT_ENABLED=true or equivalent controls in staging/prod. |
| Low | Backup route/controller files | Several .bak files are present in backend routes/controllers/services/validators. Exclude from build and clean up stale files before handover. |
| Low | Local .env files present | Local .env files exist in workspace. Ensure they are not committed and rotate any exposed credentials before deployment. |
| Low | Rendered ERD artifact not found | Prisma schema contains enough relation metadata, but a generated ERD diagram was Not Found in Current Codebase. Generate one for audit package if required. |
